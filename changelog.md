# AZS Mail Robot — История версий

---

## v3.0 (2026-06-03) — Текущая версия

### 🚀 Новые возможности
- **Автообновление** — робот сам проверяет и устанавливает новые версии с GitHub Pages
- **Настройки в отдельном файле** `config_azs.json` — при обновлении настройки не теряются
- **Защита от повреждения** — проверка SHA256 при скачивании обновлений
- **Автоматический backup** — старая версия сохраняется перед обновлением

### 🔧 Улучшения
- Код и настройки разделены (безопасность, удобство обновления)
- Поддержка любого количества АЗС с разными настройками
- Проверка обновлений раз в сутки (ночью, в 3:00)

## v2.0 (2026-05-15)

### 🚀 Новые возможности
- **Умный режим работы** — три режима мониторинга (активный/пассивный/экстренный)
- **Защита после перезагрузки** — задержка 2 часа перед началом работы
- **Обработка нескольких файлов** одновременно
- **Ожидание разблокировки файла** (если Topaz ещё пишет)

### 🔧 Улучшения
- Устранена утечка памяти (корректное освобождение ресурсов SMTP)
- Исправлена критическая ошибка с сохранением pending файлов
- Автоматическая очистка логов и архива старше 30 дней
- Настраиваемые интервалы проверки

### ⚙️ Временные настройки
- Рабочее время: 07:30–09:30 (частые проверки)
- Нерабочее время: редкие проверки
- Экстренный режим: попытки каждые 5 секунд при ошибках

---

## v1.0 (2026-05-01)

### 🎯 Базовая функциональность
- Мониторинг папки Export (файлы `SesDataExport*.xml`)
- Переименование в `AZS_Shaiba*.xml`
- Отправка по email через SMTP (Reg.ru)
- Persistent queue — сохранение неотправленных файлов
- Логирование всех действий

### ⚠️ Известные проблемы
- Только один файл за цикл
- Жёсткая пауза 5 секунд без проверки блокировки
- Утечка памяти при ошибках SMTP
- Пароль в открытом виде в скрипте
- Нет автообновления
